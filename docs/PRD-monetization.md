> 更新日期: 2026-05-28

## 1. 模块概述

积分与商业化模块负责用户身份认证、积分体系、付费授权、推荐码系统和支付流程。采用轻量级后端（Fastify + SQLite）提供 API 服务，客户端通过 HTTP 请求与后端交互，主进程作为中间层转发 IPC 调用。

---

## 2. 技术架构

### 2.1 后端服务

| 组件 | 技术 | 说明 |
|------|------|------|
| Web 框架 | Fastify | 轻量高性能 Node.js 框架 |
| 数据库 | SQLite (sql.js) | 单文件数据库，无需安装 |
| 服务器端口 | localhost:3001 | 本地开发环境 |

### 2.2 数据表

| 表名 | 说明 |
|------|------|
| `users` | 用户信息：device_id、balance、referral_code、referred_by、authorized 等 |
| `purchases` | 订单信息：trade_no、user_id、product_id、amount_cents、status 等 |
| `consumption_log` | 消费记录：point_change、timestamp 等 |

### 2.3 客户端架构

```
主进程 (points.ts)  ←→  后端 API (localhost:3001)
       ↕ IPC
渲染进程 (usePointsState.ts → PointsDisplay / PurchasePage / PaymentModal)
```

用户数据本地缓存在 `%APPDATA%/yuyin/user.json`，定期与后端同步。

## 3. 积分体系

### 3.1 积分获取

| 途径 | 积分 | 说明 |
|------|------|------|
| 首次登录 | 10,000 点 | 新用户注册即送，不含额外 +50 |
| 每日登录 | +50 点 | 第 2 天起，每日首次启动自动发放 |
| 邀请好友 | +1,000 点 | 好友填写推荐码注册，双方各得 1,000 点 |

### 3.2 积分消耗

| 用途 | 消耗 | 说明 |
|------|------|------|
| 观看直播/视频 | 100 点/小时 | 开始播放时启动计时，停止播放时暂停 |
| 自由浏览网页 | 100 点/小时 | 同上 |
| 阅读小说 | 免费 | 小说模块不消耗积分 |

### 3.3 授权状态

- **授权用户**（`isAuthorized: true`）：永久 VIP 或有效期内用户，无积分消耗
- **未授权用户**：积分消耗制，余额不足时提示购买

## 4. 推荐码系统

### 4.1 推荐码生成

- 每个用户注册时自动生成唯一 6 位推荐码（如 `A3F8K2`）
- 推荐码与用户绑定，不可更改

### 4.2 推荐流程

1. 老用户在点数抽屉中查看推荐码，点击「分享」复制推广文案
2. 推广文案包含：`鱼隐 - 工位摸鱼神器，低调看直播看小说！下载地址：https://yuyinmoyu.com 推荐码：XXXXXX`
3. 新用户首次启动时弹出欢迎弹窗，可选择填写推荐码
4. 填写推荐码后注册，推荐人与新用户各得 1,000 点

### 4.3 首次启动弹窗 (ReferralDialog)

- 检测 `%APPDATA%/yuyin/user.json` 是否存在判断是否首次启动
- 首次启动时显示欢迎弹窗，包含可选的推荐码输入框
- 推荐码为可选项，用户可跳过直接使用
- 推荐码仅在首次注册时有效，老用户无法补填

## 6. 支付系统

### 6.1 支付方式

采用支付宝个人收款码（非商户 API），用户扫码支付后手动确认。

### 6.2 支付流程 (PaymentModal)

```
创建订单 → 扫码支付 → 二次确认 → 倒计时验证 → 完成
```

**阶段 1：扫码支付 (pay)**
- 显示支付宝收款二维码（`local-assets://alipay-qr.png`）
- 显示精确支付金额（大号黄色文字）
- 提示用户支付精确金额（含随机尾数），便于核实
- 显示订单号和推荐备注后 6 位

**阶段 2：二次确认 (confirm)**
- 警告图标 + 确认提示
- 红色警告：未支付却确认将被系统记录，异常账户将被限制
- 「返回支付」/「确认已支付」双按钮

**阶段 3：验证中 (verifying)**
- 10 秒倒计时 + 进度条动画
- 倒计时结束后调用 `purchase-claim` API

**阶段 4：完成 (done)**
- 成功提示 + 自动刷新授权状态
- 1.5 秒后关闭弹窗

### 6.3 随机金额机制

每个订单生成随机金额以标识唯一订单：

```javascript
const randomCents = Math.floor(Math.random() * 99) + 1;  // 0.01 ~ 0.99
const actualAmount = productPrice + randomCents / 100;
```

例：半年卡 ¥68 → 实际支付 ¥68.37（随机尾数）

### 6.4 防白嫖措施

| 措施 | 说明 |
|------|------|
| 随机金额 | 每笔订单金额唯一，便于手动核实到账 |
| 二次确认 | 醒目警告欺诈后果，心理威慑 |
| 每日限额 | 每用户每日最多 3 次 claim 操作 |
| 异常标记 | 未支付却确认的行为被记录 |
| 验证延迟 | 确认后 10 秒倒计时，增加作弊成本 |

## 7. 主界面点数组件 (PointsDisplay)

### 7.1 组件行为

- **无活跃内容时**：左下角显示点数按钮
- **有活跃内容时**（播放/浏览中）：自动隐藏，不遮挡内容
- 已授权用户：完全隐藏

### 7.2 点数按钮

- 显示当前余额：`⚡ 1,234`
- 余额 ≤ 500 时文字变黄色警示
- 旁边有 `↻` 刷新按钮（线框样式）

### 7.3 展开抽屉

点击点数按钮展开详情抽屉：

| 元素 | 说明 |
|------|------|
| 余额显示 | `⚡ 12,345 点` + 右上角「每日登录 +50 点」小字 |
| 获取点数按钮 | 蓝色按钮，点击打开购买页面 |
| 推荐码 | 显示推荐码 + 「分享」线框按钮，点击复制推广文案 |
| 邀请提示 | 黄色文字「邀请好友下载各得 1000 点」 |

点击抽屉外任意区域自动关闭（`ClickOutsideCloser` 组件）。

### 7.4 Tooltip

鼠标悬停点数按钮时显示提示：「点击查看详情」

## 8. 自定义协议

主进程注册 `local-assets://` 自定义协议，用于在渲染进程中加载资源文件（如收款二维码图片）：

```typescript
protocol.registerSchemesAsPrivileged([
  { scheme: 'local-assets', privileges: { bypassCSP: true, stream: true, supportFetchAPI: true } },
]);
```

资源文件通过 `extraResources` 打包到 `process.resourcesPath` 目录，开发环境则从 `assets/` 目录读取。

## 9. IPC 接口

| IPC 方法 | 方向 | 说明 |
|----------|------|------|
| `points:is-first-launch` | R → M | 检测是否首次启动（user.json 是否存在） |
| `points:init` | R → M | 初始化/同步用户，可选传入推荐码 |
| `points:get-status` | R → M | 获取当前积分状态 |
| `points:start-consume` | R → M | 开始积分消耗计时 |
| `points:stop-consume` | R → M | 停止积分消耗计时 |
| `points:purchase-create` | R → M | 创建购买订单（返回 tradeNo + amount） |
| `points:purchase-status` | R → M | 查询订单状态 |
| `points:purchase-claim` | R → M | 用户确认支付，触发后端验证 |

## 11. 已知限制

1. **手动核实支付**：个人收款码无法自动确认到账，依赖用户自报 + 随机金额人工核对
2. **无商户 API**：未接入支付宝商户支付，存在白嫖风险（已通过多重措施缓解）
3. **后端未部署上云**：当前仅 localhost 运行，需部署到云服务器后才能正式上线
4. **积分同步依赖网络**：离线时积分状态无法更新
